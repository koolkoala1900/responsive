Responsive Redesign: Toronto Cupcake 🧁

PROJECT PROMPT

In this project, we picked one page of a website to analyze based on usability and accessibility standards. Then, we built the redesigned version of this webpage. Takeaways: I gained the skills necessary to analyze and identify flaws in an existing interface, create low-fidelity and high-fidelity prototypes for various screen sizes, and build a responsive website based on those prototypes.


PICKING A WEBPAGE

The website I chose is Toronto Cupcake (https://www.torontocupcake.com/)

I chose to redesign this website since the home page has poor design choices, is visually unappealing, contains accessibility issues, and has potential for lots of improvement. Also, I love cupcakes.

FINDING PROBLEMS

I started by analyzing the usability of the web page using the criteria we learned about in class: usability (e.g., including efficiency), learnability, and memorability, and think about its conceptual model. Here are my key takeaways:

Efficiency/Learnability of Menu:

1. No reason to have the menu as hamburger even on website as there is room for full menu

2. Menu has 11 tabs, making it difficult to understand what information is where especially since each tab seems to have overlapping information

Hierarchy, Layout, Font Size, and Memorability issues

1. No textual hierarchy: logo has large text but rest text is the same size, not sure where to look first

2. Banner is useless, doesn’t add any useful information to user experience

3. Cupcakes themselves are not displayed, so not sure what is being sold, besides the 3 different logos being displayed

4. Can see three two typefaces, with no aspect in common so don’t work together at all

5. Color palette not aesthetic: 3 different shades of pink, gray, black and white

Conceptual Model and Accessibility issues

1. No forced similarity: 1 column vertical grid with lots of white space and no layout hierarchy

2. Low legibility: it is difficult to read text across the width of screen

3. When website is converted to tablet or mobile size, the menu overlaps onto the content

ACCESSIBILITY

In addition to analyzing the website on the three usability guidelines, I used WebAIM WAVE to detect possible accessibility problems. Here are its key findings

1. Heading levels were skipped multiple times in the lower half of the webpage and in the menu

2. Device dependent event handler: An event handler (hamburger menu) is present that may not be accessible.

3.Suspicious link text: Link text (in the banner) contains extraneous text or may not make sense out of context.

4. Contrast Errors with the dark pink background and white text in the hamburger menu: gained a score of 4.34

My take: I heavily agree with all of these detected problems, as they lead to issues with using this website on a tablet/phone and overall contributes to difficulty in understanding the website for a wide range of users.

HIGH FIDELITY PROTOTYPES

Using Figma, I created a total of 3 high-fidelity prototypes of your screen, one for each of the screen sizes (mobile, tablet, desktop). These are based on the low-fidelity wireframes I created with one exception: I made changes to the web version by making the pictures in 4-grid layout as opposed to the 2-grid. The overflow at the bottom indicates user scrolling down.

Additionally, the annotations for the iPhone and iPad mockups are below the images

Here are the annotations for the iPhone and iPad mockups:

Page changes:

1. When user clicks any of the icons on the menu on top right

2. When user clicks go to menu button

3. When user clicks any of the social media icons

Fonts and Sizes:

1. Large elements use columns/grids of 1, 2, 3, or 4

2. Header will use h1

3. Rest of the titles “House Favorites” “About the company” and “About Us” use h3

4. When in between sizes, for example with the contact us part, spacing needed to be adjusted slightly to accommodate the form

Accessibility:

1. Menu drops directly down below the header for iPhone and iPad (usually on the right on website)

2. Cupcakes are displayed in 4-grid on website, but are displayed in 2-grid for iPad and 1-grid for iPhone

TAKEAWAYS

The redesign process taught me a lot about what it means to create a user-friendly and intuitive interface whilst still retaining the key pieces of information and content. I was shocked to see how big companies like Toronto Cupcake haven’t invested the time to create a responsive website for their customers. This process also gave me insight into the font, color, and design choices that need to be made for a company.
